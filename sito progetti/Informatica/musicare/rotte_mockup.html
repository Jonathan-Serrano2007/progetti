Ospite Rotte pubbliche — nessuna autenticazione richiesta
GET index.php
Landing page
Pagina di benvenuto. Presenta la piattaforma con sezioni «Chi siamo», prova gratuita e call-to-action per login/registrazione. Se l'utente è già loggato viene reindirizzato automaticamente a dashboard.php.
→ dashboard.php se loggato
GET POST login.php
Login utente
GET: mostra il form di accesso.
POST (email, password): verifica le credenziali, avvia la sessione ($_SESSION['utente_id'], 'utente_nome', 'utente_ruolo') e reindirizza a dashboard.php. Gestisce anche il logout tramite ?logout.
→ dashboard.php se loggato
GET POST register.php
Registrazione nuovo utente
GET: mostra il form di registrazione.
POST (nome, cognome, email, password): inserisce il nuovo utente in utenti, inizializza il record in progressi e assegna il ruolo registrato di default. Usa transazione per garantire la coerenza tra le tabelle.
→ login.php dopo il successo
Registrato Rotte per utenti autenticati — tutti i ruoli
GET dashboard.php login richiesto
Dashboard personale
Hub centrale dopo il login. Mostra nome, ruolo e lista privilegi dell'utente. Visualizza widget differenti in base al ruolo (card per esercizi base, accesso area pro, pannello admin). Supporta il logout tramite ?logout.
Privilegio: — (qualsiasi utente loggato)
→ login.php se non loggato
GET categoria_esercizi.php login richiesto
Catalogo categorie esercizi
Mostra le 4 categorie disponibili: Dettato ritmico, Ear training, Riconoscimento delle note, Tonalità. Ogni card collega a elenco_esercizi.php?cat={slug}.
Privilegio: — (qualsiasi utente loggato)
→ login.php se non loggato
GET elenco_esercizi.php?cat={categoria} login richiesto
Elenco esercizi per categoria
Lista gli esercizi della categoria richiesta tramite query string (?cat=dettato_ritmico | ear_training | riconoscimento_note | tonalita).
Verifica se l'utente ha un abbonamento attivo in abbonamenti: gli utenti senza abbonamento vedono il banner di upgrade; gli utenti Pro/Admin hanno accesso pieno.
Privilegio: svolge_esercizi_base (base) | abbonamento attivo per contenuti pro
→ categoria_esercizi.php se cat invalida
GET esercizi_base.php login richiesto
Esercizi base
Carica tutti gli esercizi con tipo_esercizio = 'base' ordinati per difficoltà crescente. Mostra gli ultimi 10 progressi dell'utente dalla tabella svolge.
Privilegio: svolge_esercizi_base (tutti i ruoli loggati)
→ login.php se non loggato
GET POST esercizio.php login richiesto
Svolgimento esercizio
GET: carica l'interfaccia interattiva di un esercizio (identificato da ?esercizio=N, N da 1 a 10).
POST (action=save_progress, exercise_id, result, time_spent): salva il risultato in svolge, ricalcola media_punti e tempo_medio_impiegato in progressi tramite UPSERT. Risponde in JSON.
Privilegio: — (qualsiasi utente loggato)
→ 401 JSON se non loggato (su POST)
Pro Rotte per utenti con abbonamento Pro (+ Admin)
GET esercizi_pro.php ruolo: pro / admin
Esercizi avanzati (PRO)
Carica gli esercizi con tipo_esercizio = 'avanzato' ordinati per difficoltà decrescente. Mostra gli ultimi 20 progressi. L'accesso è limitato ai ruoli pro e admin; gli altri vengono rimandati alla dashboard.
Privilegio: svolge_esercizi_pro
→ dashboard.php se non autorizzato
GET statistiche.php ruolo: pro / admin
Statistiche personali
Mostra le statistiche individuali: media_punti, tempo_medio_impiegato dalla tabella progressi, e lo storico completo degli svolgimenti ordinato per data.
Privilegio: visualizza_statistiche
→ dashboard.php se non autorizzato
Admin Rotte riservate agli amministratori (id_ruolo = 3)
GET POST admin_users.php SOLO admin
Gestione utenti
GET: elenca tutti gli utenti con ruolo corrente.
POST (azione=cambia_ruolo, id_utente, nuovo_ruolo): aggiorna utenti.id_ruolo.
POST (azione=elimina_utente, id_utente): elimina l'utente (con cascata su progressi, svolge, abbonamenti).
Privilegio: gestione_utenti
→ dashboard.php se non admin
GET POST admin_exercises.php SOLO admin
Gestione esercizi
GET: elenca tutti gli esercizi presenti nel db.
POST (azione=nuovo_esercizio, categoria, difficolta, tempo): inserisce un nuovo esercizio.
POST (azione=elimina_esercizio, id_esercizio): rimuove l'esercizio.
Privilegio: gestione_esercizi
→ dashboard.php se non admin
GET admin_statistics.php SOLO admin
Statistiche globali
Pannello di controllo con metriche aggregate: numero totale utenti, esercizi presenti nel catalogo, svolgimenti registrati e medie di punteggio/tempo su tutta la piattaforma.
Privilegio: visualizza_statistiche_globali
→ dashboard.php se non admin
API / JWT Endpoint REST — autenticazione tramite Bearer token
POST API/auth_api.php
Login JWT
Body JSON: { "email": "…", "password": "…" }
Risposta: { "access_token": "…", "refresh_token": "…", "expires_in": 600 }
Access token valido 10 min, refresh token 7 giorni (salvato nel campo refresh_token della tabella utenti).
200 / 401 / 405
GET POST API/permissions_api.php
Lettura permessi via JWT
Header: Authorization: Bearer <access_token>
Risposta: { "utente": {…}, "ruolo": "…", "privilegi": […] }
Decodifica il JWT, interroga ruolo_privilegi e ritorna il profilo completo delle autorizzazioni.
200 / 401 / 405
POST API/refresh_api.php
Rinnovo access token
Body JSON: { "refresh_token": "…" }
Verifica il refresh token nel DB, emette un nuovo access token (10 min) senza richiedere nuovo login.
200 / 401 / 405
